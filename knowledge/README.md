# 全局知识库

跨项目积累的创作规律、失败经验和可复用片段。

## 三层结构

1. `candidates/`：LLM 汇总输出的候选条目，允许粗糙。
2. `reviews/`：筛选文档，用中文标记保留、丢弃、合并、稍后。
3. `entries/`：正式知识库，只放用户确认过的高价值内容。

## 调用规则

- **02 Structure 及之前**：可读取 `entries/` 辅助方向判断。
- **03 Outline 及之后**：只写入（capture），不读取，避免污染当前故事的内部一致性。

## 来源

知识来自各项目的 `00_Creative_Chat_Log.md`，由用户主动触发汇总后进入候选层。
