## 一、硬指标（可量化）

### 1.1 字数规格

| 指标 | 标准 | 来源 |
|------|------|------|
| 短篇最低字数 | 8,000 字（低于直接拒） | 官方 |
| 短篇上限 | 100,000 字 | 官方 |
| 最佳短篇甜区 | 10,000 – 30,000 字 | 多位作者经验 |
| 单章字数 | 600-800 字（盐选主流） | 官方 + 多位作者经验 |
| 首次发布量（回答体） | 2,000 – 5,000 字，必须包含核心冲突 | 官方 + 作者经验 |
| 免费章节上限 | 全文的 1/3，且不超过 6,000 字 | 签约合同规定 |

### 1.2 开篇（逐级检查）

| 位置 | 必须完成 | 来源 |
|------|---------|------|
| 前 50 字 | 足够吸引人点开 | 用户资料 |
| 前 100 字 | 出现冲突/结果/异常事件；交代背景+人设+方向+基调 | 用户资料 + 作者经验 |
| 前 200 字 | 能说出"这个故事讲什么"且想看下去；拎主线+找看点 | 盐选全流程拆解 |
| 前 300-500 字 | 编辑只看这些决定是否继续审稿；必须出现核心事件 | 编辑反馈（多源反复强调） |
| 前 1,000 字 | 主角欲望 + 动机明确 | 盐选全流程拆解 |

### 1.3 节奏密度

| 指标 | 标准 | 来源 |
|------|------|------|
| 情绪波动频率 | 每 600-800 字至少 1 次情绪变化 | 用户确认（综合多源取中间值） |
| 爽点/小反转间隔 | 每 800 – 1,000 字至少 1 个 | 作者经验 + AI 社区 |
| 钩子密度 | 前 3 章（或前 3,000 字）至少 3 个钩子 | DeepSeek爆款 |
| 三章必反转 | 前 3 章内至少 1 次读者预期被打破 | DeepSeek爆款 |
| 冲突频率 | 每 2 章内至少 1 个"目标 + 阻碍"冲突 | 盐选全流程拆解 |
| 小反转周期 | 每 3 章 1 个小反转 | 作者经验 |
| 大高潮周期 | 每 10 章 1 次 | 作者经验 |
| 前 1/3 – 1/2 处 | 必须有大转折或小高潮 | 盐选全流程拆解 |

### 1.4 付费点

| 指标 | 标准 | 来源 |
|------|------|------|
| 截断位置 | 全文的 35%-45%（按比例） | 用户确认 |
| 免费最后一章 | 必须有"无法忍受不看下去"的超级悬念 | 盐选全流程拆解 |
| 付费章爽点 | 每 1-2 章内有主角阶段性胜利或反派吃瘪 | 盐选全流程拆解 |

### 1.5 平台数据门槛

| 指标 | 标准 | 来源 |
|------|------|------|
| 完读率 | ≥ 45% 进入基础分成池 | AI 社区经验 |
| 目标阅读完成率 | ≥ 55% | 平台内部指标（证券研报） |
| 付费转化率 | ≥ 10% 进入追加池 | 2026年盐选收益指南 |
| 冷启动点击率 | ≥ 12% | 平台内部指标 |
| 15 秒页面停留率 | ≥ 60% | 平台内部指标 |
| 点赞率 | ≥ 1.5%（每 100 阅读 ≥ 1.5 赞） | 平台内部指标 |
| 互动密度 | ≥ 0.8%（评论/阅读） | 平台内部指标 |

### 1.6 结构

| 指标 | 标准 | 来源 |
|------|------|------|
| 三段式 | 开头冲突、中段升级、结尾反转 | 用户资料 |
| 主线 | 单一贯穿，不设多条并行主线 | 盐选投稿攻略 |
| 场景密度 | 每章不超过 3 个场景（频繁切换破坏代入感） | 深度搜索 |

## 二、软指标（需人工判断）

### 2.1 叙述风格

| 检查项 | 标准 | 来源 |
|--------|------|------|
| 人称 | 第一人称"我"为主流 | 盐言故事叙事风格 |
| 语言 | 口语化，通俗流畅，短、平、快、爽 | 多源 |
| 句式 | 短句为主，单句超 50 字的长句比例 > 20% = 节奏风险 | 盐选叙事风格 |
| 段落 | 短段落，适配手机碎片化阅读，无大块密集文字 | 深度搜索 |
| 修辞 | 不堆砌好词好句，不用华丽辞藻 | 深度搜索 |
| 代入感 | 读者能否代入主角处境（"代入感弱"= 高频拒稿理由） | 盐选拒稿分析 |
| 文笔 | 多用动作和对话推进，减少副词、抽象表达、长篇心理描写 | 用户资料 |
| 对话标点 | 用「」（直角引号）替代 "" | 深度搜索 |

### 2.2 导语设计

| 维度 | 说明 | 来源 |
|------|------|------|
| 位置 | 独立于正文之外，不放在正文里 | 深度搜索 |
| 长度 | 4-6 句，50 字以内 | 深度搜索 |
| 公式 | 噱头 + 细节递进情绪 + 转折 + 联合正文 | 深度搜索 |
| "三有一反" | 有内涵、有悬念、有情绪、反常识 | 用户资料 |
| 类型 | 悬念型 / 反差型 / 极端事件型 | 深度搜索 |
| 禁止 | 抽象描述，必须具象化冲突 | 深度搜索 |

### 2.3 人物与冲突

| 检查项 | 标准 | 来源 |
|--------|------|------|
| 主角性格 | 强烈鲜明，一个形容词能概括 | 盐选投稿攻略 |
| 反机制 | 角色不能只会听话，必须有反抗/挣扎 | 深度搜索 |
| 优缺点 | 必须同时有优点和缺点，有成长空间 | 深度搜索 |
| 动机 | "想要什么，为什么想要"必须清晰 | 深度搜索 |
| 记忆点 | 人设带记忆点（病娇、反套路职业、极端身份） | 深度搜索 |
| 工具人 | 配角只为推动情节、没有自身动机 = 不通过 | PROJECT_RULES |
| 冲突来源 | 依赖巧合或空降信息 = 不通过 | PROJECT_RULES |
| 成长弧 | 短篇至少需要主角有一个清晰变化，起点 = 终点 = 不通过 | 盐选投稿攻略 |
| 共鸣 | 核心冲突能否让目标读者联想到自身经历 | 盐选作者管理规范 |

### 2.4 对话质量

| 检查项 | 标准 | 来源 |
|--------|------|------|
| 区分度 | 每个角色说话风格不同（外向话多、内向话少） | 深度搜索 |
| 潜台词 | 对话需有言外之意，不能全是表面意思 | 深度搜索 |
| 动作配合 | 人物说话时配动作和表情，避免全是"XX说" | 深度搜索 |
| 方言/网络梗 | 适度使用增加真实感 | 深度搜索 |
| 同质化 | 所有角色说话方式相似 = 不通过 | PROJECT_RULES |

### 2.5 付费点截断形态

| 形态 | 说明 | 来源 |
|------|------|------|
| 转折 | 剧情出现重大反转 | 用户资料 |
| 情绪 | 情绪高涨时戛然而止 | 用户资料 |
| 打断 | 对话或行动突然中断 | 用户资料 |
| 揭秘 | 关键答案呼之欲出但暂不揭示 | 用户资料 |
| 抉择 | 主角面临二选一的重要选择 | 用户资料 |

### 2.6 情绪曲线

| 检查项 | 标准 | 来源 |
|--------|------|------|
| 情绪节拍 | 每章标注读者应感受的情绪类型和强度（1-10） | 深度搜索 |
| 曲线模型 | 铺垫 → 高潮 → 过渡 的循环 | 深度搜索 |
| 避免连续高潮 | 持续加速会导致情绪麻木 | 深度搜索 |
| 后段减速 | 跑到 2/3 处需开始缓冲，不能持续加速到结尾掉悬崖 | 深度搜索 |

### 2.7 完读率保障

| 检查项 | 标准 | 来源 |
|--------|------|------|
| 后半段掉速 | 付费后连续 > 2,000 字纯过渡（无冲突/新信息/情绪变化）= 风险 | 完读率机制推断 |
| 结尾升华 | 收尾需交代人物结局 + 升华主题，仓促收尾 = 不通过 | 盐选全流程拆解 |

### 2.8 付费性综合

| 检查项 | 标准 | 来源 |
|--------|------|------|
| 付费冲动 | 读到免费末尾时是否"必须知道后面发生了什么"？无 = 不通过 | 盐选拒稿分析 |
| 前期拖沓 | 前 3,000 字内有多余背景铺垫？有 = 需删减 | 盐选拒稿分析 |
| 信息差持续 | 免费部分已把核心信息全部泄露？是 = 无付费动力 | 综合推断 |

## 四、爆款特征（非必须但显著提升）

### 4.1 选题公式

**爆款 = 50% 选题 + 20% 标题封面 + 20% 文案内容 + 10% 其他**

三大新人优先题材：
1. 职场黑幕（财务造假/性骚扰掩盖）
2. 婚姻博弈（彩礼纠纷/伪造绝症）
3. 阶层碰撞（外卖员 VS 豪宅客户）

全平台热门题材：
- 都市向（职场 + 家庭）— 优先
- 悬疑向（刑侦 + 爱情）— 优先
- 虐恋（张力 + 宿命感）
- 脑洞设定（穿越/时空）
- 亲情（治愈/扶持）

共性公式：**情感冲突 × 现实痛点 × 悬念**

饱和预警：言情甜文同质化严重，需强差异化。

### 4.2 标题公式

高点击率模式：
- 具体数字 + 身份反差 + 悬念词
- 例：《月薪3千到年薪百万：我在互联网公司当"背锅侠"的3年》
- 例：《重生之我在霸总婚礼上抢走白月光》

### 4.3 爆款叙事特征

| 特征 | 说明 |
|------|------|
| 开篇即冲突 | 第一章出现核心事件 |
| 三章必反转 | 读者持续被拉住 |
| 每章留钩子 | 碎片化阅读下每章结尾有"点开下一章的冲动" |
| 人设带记忆点 | 病娇、反套路职业、极端身份 |
| 具象化冲突 | 禁用"人性的深渊"，要用"为了活命他吃了自己养的宠物狗" |
| 来回拉扯 | 情感场景不直接达成，要有暧昧感的推进 |
| 铺垫变因果 | 所有巧合都有前因，设好铺垫和伏笔 |

### 4.4 爆款开头模式

- 日常场景 + 反常线索（在熟悉中制造陌生感）
- 对比/反差开局（身价过亿却住贫民窟）
- 极端事件直入
- 强情绪冲击（第一句就带情绪）

### 4.5 读者心理抓手

| 抓手 | 说明 |
|------|------|
| 爽文核心 | 满足现实中难以实现的权力欲、优越感、复仇幻想 |
| 情绪产品 | 代入感 > 文笔质量 |
| 吃瓜心理 | 强冲突契合用户"找乐子"的需求 |
| 信息差 | 读者知道主角不知道的事，或反过来 |

## 五、常见失败模式（反向检查清单）

### 5.1 拒稿高频原因

| 失败模式 | 具体表现 | 来源 |
|----------|----------|------|
| 开头无冲击力 | 前 500 字没有抓人元素，编辑直接毙 | 编辑反馈 |
| 节奏拖沓 | 长篇大论介绍背景/设定 | 编辑 + 作者经验 |
| 套路化 | 梗太老，剧情把控差 | 编辑反馈 |
| 主线不清晰 | 叙事混乱，无关文字过多 | 编辑反馈 |
| 付费感不足 | 读者在截断点没有付费冲动 | 编辑反馈 |
| 代入感弱 | 文笔风格、剧情逻辑经不起推敲 | 编辑反馈 |
| 风格不符 | 非知乎体（过于文学化/过于网文化） | 大量拒稿案例 |
| 题材饱和 | 言情甜文同质化，缺乏差异化 | 盐选拒稿高频原因 |

### 5.2 AI 写作常见翻车

| 失败模式 | 具体表现 |
|----------|----------|
| 提纲腔 | 情节像大纲翻译，缺乏细节和场景感 |
| AI 腔 | "首先、其次、值得注意的是"等套路表达 |
| 翻译腔 | 英语语法直译，从句嵌套，汉语灵动性丧失 |
| 结构化过强 | 明显的"总分总"、markdown 式列举 |
| 情绪过于均匀 | 缺乏高低起伏，每段情绪强度类似 |
| 对话同质化 | 所有角色说话风格相同 |
| 抽象化 | 用概括代替具体细节（"充满了机遇与挑战"） |
| 过于安全/中庸 | 缺乏尖锐性和个人色彩 |

### 5.3 结构性失败

| 失败模式 | 具体表现 |
|----------|----------|
| 前提因果链不通 | "为什么非要这么做""为什么不能更简单"答不上来 |
| 转折靠巧合 | 关键剧情依赖偶然事件或空降信息 |
| 场景没立住 | 描写不够让读者形成画面感 |
| 连续高潮 | 持续加速没有缓冲，情绪麻木 |
| 虎头蛇尾 | 开头精彩但后半段质量下滑 |
| 情绪空转 | 连续多章无情绪递进，读者走神 |
| 便利型推进 | 关键转折无前置支撑 |
| 角色工具化 | 配角只负责送信息/送背刺/送反转 |

## 六、去 AI 味检查清单

### 6.1 结构与表达层

| 检查项 | 具体做法 | 来源 |
|--------|---------|------|
| 结构化词 | 删除"首先/其次/最后"，替换为语气词或直接行动 | AI 社区 |
| AI 高频短语 | 删除"值得注意的是""不难发现""总的来说""不禁""竟然" | AI 社区 |
| 对称结构 | 避免"A...B...C..."的三段并列，用不对称节奏 | writing-style skill |
| 长短句混搭 | 避免全是相同节奏的中等长度句子 | writing-voice skill |
| 副词清理 | 删除不必要的副词（"非常""极其""显然"） | writing-voice skill |

### 6.2 中文小说专属（来自 chinese-novelist skill）

| 检查项 | 具体做法 | 来源 |
|--------|---------|------|
| 过度修饰形容词 | 删减"璀璨""瑰丽""绚烂""浩瀚""深邃"等 AI 常用词堆砌 | chinese-novelist |
| 四字格律 | 打破"心潮澎湃""热血沸腾""波澜壮阔"等陈词滥调，换成具体动作或感受 | chinese-novelist |
| 抽象陈述 | "心中涌起复杂的情感" → 改为具体动作/对话/身体反应 | chinese-novelist |
| 书面语套话 | 人物对话要有个性，避免所有角色都说"标准普通话" | chinese-novelist |
| 节奏感 | 长句和短句交替，避免句式单调 | chinese-novelist |
| 细节具象化 | 用具体的视觉/听觉/嗅觉细节替代笼统描述 | chinese-novelist |

### 6.3 内容层

| 检查项 | 具体做法 | 来源 |
|--------|---------|------|
| 具体 > 抽象 | "充满了机遇与挑战" → 写出具体是什么机遇什么挑战 | AI 社区 |
| 第一人称感受 | 加入个人化的感受和反应，而非旁观者式叙述 | AI 社区 |
| 口语化 | 加入口语化表达，方言/网络梗适度使用 | 盐选风格要求 |
| 人工修改比例 | AI 生成后需人工修改至少 30% | AI 社区经验 |
| 过于安全/中庸 | 检查是否缺乏尖锐性和个人色彩 | 深度搜索 |

## 七、AI 辅助写作流程建议（来自 AI 社区）

1. **选题调研**：用推理模型分析市场热门题材
2. **大纲构建**：人物设定 + 故事框架 + 情节概要 + 描写手法
3. **内容生成**：按章节生成，指令遵循能力强的模型
4. **去 AI 化润色**：人工修改至少 30%（见第六节检查清单）
5. **情绪节拍校准**：每章标注情绪类型和强度（1-10），确保曲线有起伏

**修改方向**：这是最后一道关。原有 5 项检查故事完整性，新增 11 项做盐选全维度终审。如果前面各阶段压力测试都做了，这里应该只有少量微调，不会出现"推翻重来"。

---

### 阶段压力测试总览

```
成本低 ◄──────────────────────────────────────────────────► 成本高
改方向      改前提      改逻辑      改结构      改大纲      改写法      改文本

Intake    → Killer    → Premise   → Eval      → Structure → Outline   → ScenePT  → Proto    → Expansion → Review
原有：       原有：       原有：       原有：       原有：       原有：       原有：       原有：       原有：       原有：
 (无)        4问评分      5问校验      4项评估      5项骨架      5项功能      5维打分      7项检查      参照表       5项复核
新增：       新增：       新增：       新增：       新增：       新增：       新增：       新增：       新增：       新增：
 红线/题材    6项适配      4项人物      5项商业      6项盐选      9项节奏      5项盐选      6项风格      7项单章      11项终审
```

**核心逻辑**：
1. 每个阶段 = 原有检查 + 盐选适配检查，不是替换而是叠加。
2. 每个阶段只检查它该检查的东西。不在大纲阶段纠结文笔，不在扩写阶段才发现前提不通。
3. 越早发现问题，修复成本越低。Intake 改方向只需换一句话，到 Expansion 改就要重写整场。
