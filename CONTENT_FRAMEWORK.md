# 内容框架（Claude Code 版）

本文件只定义内容写法、节奏控制与阅读体验要求。

它不负责流程推进，也不负责文件管理。

详细写法指导按阶段拆分：
- Structure 阶段起加载：`CONTENT_FRAMEWORK_structure.md`（人物与驱动、冲突与推进）
- Prototype 阶段起加载：`CONTENT_FRAMEWORK_writing.md`（开篇硬指标、节奏密度、Prototype 标准、正文扩写、结尾回收）

## 1. 核心写法目标

### 1.1 快，但不能空
允许：
- 开头快
- 冲突早
- 节奏紧
- 中段不断气
- 结尾有回收感

但不能把"快"写成：
- 事件清单
- 剧情提要
- 短视频文案腔
- 只有结果，没有过程
- 只有反转，没有支撑

### 1.2 让读者贴着人物往前走
正文尽量让读者贴着人物当下反应往前走，而不是站在场外听作者复述发生了什么。

优先写出：
- 人物此刻想要什么
- 人物先感受到什么
- 人物因此做了什么反应
- 这一步让局势发生了什么变化

### 1.3 盐选感不是廉价刺激
要保留快节奏和阅读钩子，但不依赖：
- 强行反转
- 模板化狠话
- 工具人角色
- 空降信息
- 临时改规则
- 只靠更惨、更炸硬推阅读

### 1.4 情绪要落到场景里
不要只写"她很绝望""他很愤怒"。

优先把情绪落在：
- 动作
- 停顿
- 视线
- 说话方式
- 身体反应
- 当下选择
