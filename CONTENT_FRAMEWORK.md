# 内容框架（Claude Code 版）

本文件只定义内容写法、节奏控制与阅读体验要求。

它不负责流程推进，也不负责文件管理。

## 1. 核心写法目标

### 1.1 快，但不能空
允许：
- 开头快
- 冲突早
- 节奏紧
- 中段不断气
- 结尾有回收感

但不能把“快”写成：
- 事件清单
- 剧情提要
- 短视频文案腔
- 只有结果，没有过程
- 只有反转，没有支撑

### 1.2 让读者贴着人物往前走
正文尽量让读者贴着人物当下反应往前走，而不是站在场外听作者复述发生了什么。

优先写出：
- 人物此刻想要什么
- 人物先感受到什么
- 人物因此做了什么反应
- 这一步让局势发生了什么变化

### 1.3 盐选感不是廉价刺激
要保留快节奏和阅读钩子，但不依赖：
- 强行反转
- 模板化狠话
- 工具人角色
- 空降信息
- 临时改规则
- 只靠更惨、更炸硬推阅读

### 1.4 情绪要落到场景里
不要只写“她很绝望”“他很愤怒”。

优先把情绪落在：
- 动作
- 停顿
- 视线
- 说话方式
- 身体反应
- 当下选择

## 2. 开篇要求

### 2.1 第一屏就要有事发生
开头优先落在：
- 冲突现场
- 反常瞬间
- 羞辱或压迫节点
- 关系撕裂点
- 高风险选择前一秒
- 明显不对劲的细节

### 2.2 前三段尽快回答四件事
- 主角是谁
- 当前最刺人的矛盾是什么
- 谁在制造阻力
- 为什么必须继续看下去

### 2.3 避免假钩子
以下开头常见但容易失真：
- 只靠狠话撑气势
- 只抛谜面，不给局面
- 只有大信息，没有人物痛感
- 只有一句“你们都被骗了”吊着

### 2.5 盐选开篇硬指标

以下是字数级的硬要求，不是"尽量"，是"必须"：

| 位置 | 必须完成 |
|------|---------|
| 前 50 字 | 出现反差、极端事件或关系撕裂——不是铺垫，是事件已经发生 |
| 前 100 字 | 冲突已发生（不是"即将发生"）；背景、人设、方向、基调已交代 |
| 前 200 字 | 读者能说出"这个故事讲什么"且想看下去 |
| 前 300 字 | 编辑判生死的范围——核心事件必须已出现 |
| 前 1,000 字 | 主角欲望 + 动机明确 |

如果开篇不满足以上任一条，不进入下一阶段，先回修开篇。

## 3. 人物与驱动

### 3.1 主角必须有清晰想要
至少要能说清：
- 现在最想要什么
- 为什么现在必须行动
- 如果不动，会失去什么

### 3.2 对手必须形成真实阻力
不要只让对手负责骂人、放狠话和给主角当打脸对象。

### 3.3 配角不能只是工具
关键配角至少要具备一项：
- 自己的利益立场
- 自己的误判
- 自己的恐惧
- 自己的欲望

### 3.4 人物反应比作者说明重要
与其解释“她很崩溃”，不如写：
- 她第一反应想藏什么
- 她为什么没立刻回嘴
- 她做了什么不体面的动作
- 她哪一步反应让局势变得更糟或更狠

## 4. 冲突与推进

### 4.1 每次推进都要让局势发生变化
每一小段推进后，读者最好能感受到至少一种变化：
- 更危险
- 更难忍
- 更接近真相
- 更接近翻盘
- 关系站位改变
- 主角处境被重新定义

### 4.2 升级必须可感知
好的升级不是更吵，而是：
- 代价更高了
- 选择更难了
- 退路更少了
- 前面的误判开始反噬

### 4.3 信息要分层揭露
不要一口气把真相讲完。

至少分成：
- 当前必须知道的一层
- 现在只该暗示的一层
- 后面翻开更值的一层

### 4.4 30% 左右通常要有可感知的大翻面
但必须挣得出来，不能靠：
- 空降答案
- 临时改规则
- 没铺垫的巨大翻面
- 只震惊、不改局势

### 4.5 盐选节奏密度

以下是节奏层的硬要求：

| 指标 | 标准 |
|------|------|
| 情绪变化频率 | 每 600-800 字至少 1 次明确的情绪转折（不是慢递进，是可感知的变化） |
| 爽点/小反转间隔 | 每 800-1,000 字至少 1 个 |
| 钩子密度 | 前 3 章（或前 3,000 字）至少 3 个钩子 |
| 三章必反转 | 前 3 章内至少 1 次打破读者预期 |
| 每章结尾 | 必须有"翻下一页的冲动"——不是总结，是悬念或情绪截断 |
| 情绪类型 | 愤怒、委屈、爽感交替轰炸，不是"感触良多"式的平铺 |
| 对话标点 | 用「」不用""；每个角色说话风格必须不同，有潜台词 |

"太平"的典型信号：连续 1,000 字以上没有情绪转折、没有新信息、没有局势变化。

## 5. Prototype 的标准

Prototype 不是缩水大纲，而是读感验证。

重点检查：
- 开头是否够快
- 冲突是否一路推进
- 中段是否掉速
- 反转是否成立
- 结尾是否有情绪回收
- 人物是否开始工具化
- 语言是否滑向提纲 prose
- 哪一段一进正文最容易写空

成立标准不是“故事讲完了”，而是：
- 读起来像故事，不像梗概
- 人物已经开始活起来
- 节奏能带着人往前
- 关键转折不是作者硬掰
- 正文放大后有成立空间

## 6. 正文扩写

### 6.1 优先写成场景
扩写时优先呈现：
- 场景压力
- 人物反应
- 行动后的局势变化

### 6.2 常见失真信号
- outline 翻译腔
- 事情很多，但场景没立住
- 对话同质化
- 只写结果，不写过程
- 所有人都在说模板狠话
- 读者始终站在场外

### 6.3 失真后怎么处理
如果已经出现明显失真，不要继续堆正文。

先判断问题属于：
- 扩写方式
- 大纲拆分
- 结构设计
- 人物动机

然后回修对应层级。

### 6.4 用户反馈处理协议

当用户说"太平"、"没劲"、"不够狠"、"不像盐选"或类似反馈时：

**第一步：不解释、不辩护**
- 不说"其实已经有…"
- 不说"这里的设计意图是…"
- 不微调原版然后问"这样好些吗"
- 用户的审美判断不需要被评判对不对

**第二步：立即诊断是哪层太平**

| 层级 | 判断标准 |
|------|---------|
| 开篇太平 | 前 50 字没有极端事件，前 100 字冲突还没发生 |
| 节奏太平 | 情绪变化间隔超过 800 字，连续段落无新信息 |
| 情绪太平 | 烈度不够——停在"感触""难过"，没到"愤怒""委屈""爽" |
| 对话太平 | 模板化、角色区分度低、没有潜台词、全是表面意思 |
| 场景太平 | 只有叙述没有现场感，读者站在场外看而不是身在其中 |

**第三步：给出至少 3 个更狠的替代版本**
- 每个版本必须比原版在情绪烈度上明显升级
- 不是微调，是重写对应段落
- 标注每个版本的升级点在哪

**第四步：让用户选，不替用户决定**

"太平"等同于质量未通过。不允许解释后继续用原版，必须提供替代方案后由用户选择。

## 7. 结尾与回收

结尾不只是公布结果，还要让读者感到“结账了”。

优先回收：
- 前文埋下的情绪债
- 关键关系的站位变化
- 主角最初想要的东西是否真的得到了
- 代价有没有被认真兑现
