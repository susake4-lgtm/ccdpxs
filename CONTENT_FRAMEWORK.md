# 内容框架（Codex 版）

本文件只定义内容写法、节奏控制与阅读体验要求。

它不负责流程推进，也不负责文件管理。

## 1. 核心写法目标

### 1.1 快，但不能空
允许：
- 开头快
- 冲突早
- 节奏紧
- 中段不断气
- 结尾有回收感

但不能把“快”写成：
- 事件清单
- 剧情提要
- 短视频文案腔
- 只有结果，没有过程
- 只有反转，没有支撑

### 1.2 让读者贴着人物往前走
正文尽量让读者贴着人物当下反应往前走，而不是站在场外听作者复述发生了什么。

优先写出：
- 人物此刻想要什么
- 人物先感受到什么
- 人物因此做了什么反应
- 这一步让局势发生了什么变化

### 1.3 盐选感不是廉价刺激
要保留快节奏和阅读钩子，但不依赖：
- 强行反转
- 模板化狠话
- 工具人角色
- 空降信息
- 临时改规则
- 只靠更惨、更炸硬推阅读

### 1.4 情绪要落到场景里
不要只写“她很绝望”“他很愤怒”。

优先把情绪落在：
- 动作
- 停顿
- 视线
- 说话方式
- 身体反应
- 当下选择

## 2. 开篇要求

### 2.1 第一屏就要有事发生
开头优先落在：
- 冲突现场
- 反常瞬间
- 羞辱或压迫节点
- 关系撕裂点
- 高风险选择前一秒
- 明显不对劲的细节

### 2.2 前三段尽快回答四件事
- 主角是谁
- 当前最刺人的矛盾是什么
- 谁在制造阻力
- 为什么必须继续看下去

### 2.3 避免假钩子
以下开头常见但容易失真：
- 只靠狠话撑气势
- 只抛谜面，不给局面
- 只有大信息，没有人物痛感
- 只有一句“你们都被骗了”吊着

## 3. 人物与驱动

### 3.1 主角必须有清晰想要
至少要能说清：
- 现在最想要什么
- 为什么现在必须行动
- 如果不动，会失去什么

### 3.2 对手必须形成真实阻力
不要只让对手负责骂人、放狠话和给主角当打脸对象。

### 3.3 配角不能只是工具
关键配角至少要具备一项：
- 自己的利益立场
- 自己的误判
- 自己的恐惧
- 自己的欲望

### 3.4 人物反应比作者说明重要
与其解释“她很崩溃”，不如写：
- 她第一反应想藏什么
- 她为什么没立刻回嘴
- 她做了什么不体面的动作
- 她哪一步反应让局势变得更糟或更狠

## 4. 冲突与推进

### 4.1 每次推进都要让局势发生变化
每一小段推进后，读者最好能感受到至少一种变化：
- 更危险
- 更难忍
- 更接近真相
- 更接近翻盘
- 关系站位改变
- 主角处境被重新定义

### 4.2 升级必须可感知
好的升级不是更吵，而是：
- 代价更高了
- 选择更难了
- 退路更少了
- 前面的误判开始反噬

### 4.3 信息要分层揭露
不要一口气把真相讲完。

至少分成：
- 当前必须知道的一层
- 现在只该暗示的一层
- 后面翻开更值的一层

### 4.4 30% 左右通常要有可感知的大翻面
但必须挣得出来，不能靠：
- 空降答案
- 临时改规则
- 没铺垫的巨大翻面
- 只震惊、不改局势

## 5. Prototype 的标准

Prototype 不是缩水大纲，而是读感验证。

重点检查：
- 开头是否够快
- 冲突是否一路推进
- 中段是否掉速
- 反转是否成立
- 结尾是否有情绪回收
- 人物是否开始工具化
- 语言是否滑向提纲 prose
- 哪一段一进正文最容易写空

成立标准不是“故事讲完了”，而是：
- 读起来像故事，不像梗概
- 人物已经开始活起来
- 节奏能带着人往前
- 关键转折不是作者硬掰
- 正文放大后有成立空间

## 6. 正文扩写

### 6.1 优先写成场景
扩写时优先呈现：
- 场景压力
- 人物反应
- 行动后的局势变化

### 6.2 常见失真信号
- outline 翻译腔
- 事情很多，但场景没立住
- 对话同质化
- 只写结果，不写过程
- 所有人都在说模板狠话
- 读者始终站在场外

### 6.3 失真后怎么处理
如果已经出现明显失真，不要继续堆正文。

先判断问题属于：
- 扩写方式
- 大纲拆分
- 结构设计
- 人物动机

然后回修对应层级。

## 7. 结尾与回收

结尾不只是公布结果，还要让读者感到“结账了”。

优先回收：
- 前文埋下的情绪债
- 关键关系的站位变化
- 主角最初想要的东西是否真的得到了
- 代价有没有被认真兑现
