# CONTENT_FRAMEWORK

## 文件职责

本文件是共用内容标准的总入口，只负责定义“母题为什么成立、如何变体、平台执行如何分开、哪些层只作参考”。

本文件不负责：

- 阶段流推进
- 文件管理
- 下游平台的细写规则
- 可脚本化的检查细节
- Sequence / Scene 之类的派生结构

本文件的定位是上游 V1 / V1.1 成果在下游的共用标准层，不是下游拍脑袋规则汇总。

## 上游来源

本文件依据以下上游成果重写：

- `V1上游标准-总览.md`
- `V1上游标准-总纲.md`
- `V1母题系统骨架-v1.md`
- `V1母题变体规则包.md`

其中当前可直接信任的基线是：

- `V1母题系统骨架-v1.md`
- `V1母题变体规则包.md`

上游总览与总纲用于确认交付顺序、权威边界和下游闭环口径。

## 共用母题成立标准

共用母题是否成立，先看四个骨架是否齐全：

- 冲突骨架
- 关系骨架
- 情绪骨架
- 推进机制

一个母题要能作为共用标准，至少要满足这些条件：

- 主角为什么会进入这场冲突，说得清
- 读者为什么天然站主角，说得清
- 对方为什么会站错队、失配或误判，说得清
- 主角后续为什么能切断、反打、换打法，说得清
- 这个冲突能够持续做变体，而不是只能依赖单个情节

共用母题的目标不是统一题材表面，而是保住底层发动机。

## 母题变体总原则

母题变体只允许在不伤发动机的前提下变换外壳。

可变的部分包括：

- 背景
- 人设
- 职业
- 世界规则
- 信息差或设定差

不可变的部分包括：

- 主角为何受伤或受压
- 对方为何会站错队
- 读者为何会站主角
- 主角为何有理由切断、反打或换打法

变体时优先抓的是骨架，不是题材词。题材词只是表层包装，不能代替冲突本体。

上游 V1 母题变体规则包当前作为可信基线使用，后续可以补样修订，但不应先退回到下游猜测写法。

## 平台执行总原则

平台执行必须和共用母题分开，不能混用。

当前原则是：

- 母题可以共用
- 执行不能混用
- 标题、开头、文风、结构、审核都按目标平台分别处理
- `target_platform` 由下游或用户输入指定，不在本文件中自动判定

当前 V1 的平台规则包作为执行可信基线使用，但它们属于平台层，不是共用母题层。

本文件只保留平台执行分层的原则，不写下游平台的细节化写法，也不把平台经验硬塞回母题标准。

## 参考层边界

以下内容只作参考层，不作为本文件的硬门禁：

- 事件弧
- 逆向骨架
- V2 相关治理思路

参考层可以用于：

- 结构判断
- 大纲整理
- 扩写提示
- 复核时的辅助阅读

但参考层不得直接变成：

- 正式输入锚点
- 硬性准入门禁
- 脚本化检查细则
- 平台混用的借口

如果参考层与共用母题标准冲突，以共用母题标准和当前可信基线为准。

## 更新口

当上游出现新的稳定 V1 / V1.1 成果时，本文件只按以下方向更新：

- 先确认是否属于共用母题层、平台执行层或参考层
- 共用母题层优先吸收稳定成果
- 平台执行层只吸收平台自身的稳定包
- 参考层保持 docs-only，不抢占硬约束位置

更新本文件时，应继续保留下游执行壳，不把共用标准改成流程说明，也不把参考层抬成门禁层。
